Costume Enchant System

  • You can enchant a Costume with a Costume Enchant by double clicking on the item as if it were a card.
  • You can enchant a Costume made by Miles the Milliner. It will then have a bonus enchant parameter of "Costume"
  • Garments which have been enchanted through the Hat Genie can receive Enchantment Stones.
  • Costumes that have been enchanted becomes account bound and cannot be traded. It will once again be tradable if the enchant is removed.
  • Costumes that go on multiple headgear slots can be slotted with the respective stones. Eg. Upper-Mid can be slotted with [U] and [M] Stones.
  • You can freely remove the Costume Enchant from the costume with a decarder NPC. It is a safe process, so you will keep the Costume Enchant.
  • All Costume Enchants are account-bound.
  • Some Costume Enchants are found from field and dungeons, but Onyxes are obtained via crafting recipes from polished stones.
  • Costume Enchants cannot be put into Signed Costumes. This is a technical limitation as Signing occupies the same slots that Costume Enchants use.
  • Bubble Gum does not impact the drop rate of these items.
  • They're affected by @aloottype +6 when dropped.

Pity System

When a player defeats the number of monsters they reasonably expect to kill (100 divided by the item drop rate) for stone obtainment, every additional monster they defeat past this number will increase the drop rate for the stone following this formula: New Drop Rate = Drop Rate + (Drop Rate * Number of kills past threshold)

For example, where a Stone's drop is 0.20%

100 / 0.20 = 500

Then every kill past kill 500 would increase the drop rate of the stone by 0.20% until the item dropped. This value can increase all the way up to 100% drop chance.

  • Pity value for the most recent Stone kill can be viewed via the @pity command. (Or use @pitycount to show pity value everytime a relevant monster is killed.)
  • Pity is counted Per Stone, ie. Stones that come from two mobs increase pity value on the combined kills of the two mobs.
  • Pity value is stored Per Character, ie. the count is not shared for chars in the same account, each will have their own counters.
  • Regarding some mobs that summon other mobs, it would be best to assume that summoned mobs won't increase Pity Count.

Pity Reset

  • Upon dropping a Stone, regardless of the Pity Count and regardless if you picked it up from the floor or via autoloot, the count will revert to 0.
    Ie. If you're farming and did not have autoloot on and it drops to the ground and you left it, your Pity Count resets to 0.
  • Pity Count persists for any mob that drops stones (and you haven't gotten a stone yet) as long as you don't Log Out or Change Character.
  • Upon Logging Out or Changing Characters, only the most recent five Mobs' Pity counts persist.

Theoretical Values

For reference,
  • the mean (a.k.a. expected value) of the number of target monsters that a player needs to defeat to obtain a stone has been decreased from
100 / Drop Rate (= Pity Threshold)
without the pity system to approximately (and slighter larger than)
100 * (1 - 1/e) / Drop Rate (= 63.2% * Pity Threshold)
with the pity system, which is a substantial increase in effective drop rate,
  • the median of the number of target monsters that a player needs to defeat to obtain a stone, which does not change regardless of whether the pity system was in place, is approximately
100 * ln(2) / Drop Rate (= 69.3% * Pity Threshold)
(i.e. for 50% of the time, a player can expect to get a stone before defeating this number of monsters), and
  • the cumulative probability of getting a stone before reaching the pity threshold is approximately
1 - 1/e (= 63.2%)
(i.e. for 63.2% of the time, a player can expect to get a stone before getting pitied).

Stone Upgrade System

Vending Recipes

Speaking with Judd at (main_office 306, 292)Copied!, you'll be able to sell the Costume Enchant Stones for crafting materials. These materials can then be used to buy other Costume Enchants. It is safe to vend Costume Enchants in bulk to the NPC as it will always try to get the best outcome possible. If both Enchant Stones are identical, the reward will be downgraded as shown below. For example:

  • Selling 2x  Critical Amber [L] to the NPC will yield a  Honed Jade.
  • Depreciated as per Patch 177 adjustment. Now: Selling  Critical Amber [L] +  Ranged Amber [L] will yield a  Honed Amber as the two items used are different.
  • You cannot mix the types (e.g. you cannot sell an Amber and a Jade enchant).
  • In summary, the only scenario where you get a different honed stone type than what you brought to the NPC is when you bring in 2 of the exact same stone.
  • Those who exchanged stones prior to Patch 177 will receive 'Generic' labeled Stones. These function as exchange stand-ins. Eg.  Generic Amber +  Ranged Amber [L] =  Honed Amber
Output Recipe
1x  Honed Lapis 2x of same Lapis type
1x  Honed Lapis 2x of same Jade type
1x  Honed Jade 2x of same Amber type
1x  Honed Lapis 2x of different Lapis type
1x  Honed Jade 2x of different Jade type
1x  Honed Amber 2x of different Amber type
